## 일단 한 번 돌려본다

새로운 도구를 배울 때 어떤 방식이 가장 효과적이던가요? 저는 "이론을 먼저 이해한다"보다 "일단 한 번 돌려본다"가 훨씬 빨리 감이 잡히더군요.

PaperClip은 본질적으로 "AI 회사 운영 체제"입니다. 운영 체제라는 건 직접 돌려보지 않으면 감이 오지 않습니다. 그래서 이 단계의 목표는 딱 하나, **일단 PaperClip을 띄우고 브라우저에서 온보딩 마법사가 열리는 순간까지 가는 것**입니다.

개념을 먼저 이해하려고 애쓰지 마세요. 여러분이 할 일은 터미널을 열고, 명령어 한 줄을 복사해 붙여넣고, 엔터를 치고, 브라우저에서 첫 화면이 열리는 모습을 확인하는 것뿐입니다. 이 작은 성공 경험이 앞으로 한 시간 동안 여러분을 끌고 갈 동력이 됩니다.

## 한 줄 명령 속에서 벌어지는 일

설치는 `npx paperclipai onboard --yes` 한 줄로 끝납니다. 그런데 이 한 줄이 꽤 많은 일을 합니다. 내부적으로는 홈 디렉토리 밑에 `~/.paperclip/instances/default/` 구조를 만들고, 내장 PostgreSQL(PGlite)을 초기화하고, JWT 시크릿(로그인 토큰을 만들 때 쓰는 비밀 키)을 자동 생성하고, API 서버(포트 3100)와 React UI를 동시에 띄웁니다.

`--yes` 플래그는 설치 중 모든 확인 질문에 자동으로 "예"를 답하게 만듭니다. 덕분에 한 번의 엔터로 설치가 끝납니다. 편하지요?

## 직접 실행

터미널을 열어 주세요. 작업 디렉토리는 어디여도 상관없습니다. PaperClip 데이터는 홈 디렉토리에 저장되므로, 어느 폴더에서 실행해도 결과는 똑같습니다.

:::note[터미널 기초 — 처음이라면 먼저 읽기]

"터미널을 열어 주세요"라는 한 줄 앞에서 손이 멈추는 분이 있을 겁니다. 당연한 일입니다. 평소에 마우스로만 컴퓨터를 쓰던 분에게 검은 화면에 글자만 있는 터미널은 낯설지요. 기초부터 한 번만 짚고 가면 됩니다.

**터미널 여는 법**

- **macOS**: `⌘ + Space`로 Spotlight(스팟라이트, 맥의 검색창)를 띄우고 "터미널" 또는 "Terminal"이라고 입력한 뒤 엔터를 누르세요. 바로 검은 창이 뜹니다.
- **Windows(WSL 권장)**: `Win + R`을 눌러 실행창을 연 뒤 `wsl`이라고 입력하고 엔터를 누르면 됩니다. WSL(Windows Subsystem for Linux, 윈도우 안에서 리눅스를 돌리는 기능)이 설치되어 있어야 하며, 이 교재는 WSL 환경을 전제로 안내합니다. PowerShell이나 cmd로도 가능하지만, 명령어 호환성이 더 좋은 WSL을 추천합니다.

**명령어 복사·붙여넣기**

본문의 코드블록을 마우스로 드래그해 선택한 뒤,

- **macOS**: `⌘C`로 복사, 터미널 창을 클릭해 활성화한 뒤 `⌘V`로 붙여넣기.
- **Windows(WSL)**: `Ctrl + C`로 복사, 터미널 창에서 `Ctrl + Shift + V`(또는 마우스 오른쪽 버튼)로 붙여넣기. WSL에서 `Ctrl + C`는 "실행 중인 명령 강제 종료"라는 다른 뜻이 있어서, 붙여넣기에는 `Shift`를 함께 눌러야 합니다.

**"끝난 건가, 아직인가" 구분하기**

엔터를 친 뒤 글자가 계속 쏟아져 나오면 아직 명령이 실행 중입니다. 기다려 주세요. 화면이 잠잠해지고 맨 아래 줄에 `$` 또는 `%` 같은 기호(이걸 **프롬프트**(Prompt, 명령 입력 대기 표시)라고 부릅니다)가 다시 나타나면, 그때가 "명령이 끝났고 다음 입력을 받을 준비가 되었다"는 신호입니다. 이 신호가 보이기 전에는 추가 명령을 입력하지 말고 가만히 둬 주세요.

**오류가 떴을 때**

빨간 글씨나 `Error:`로 시작하는 줄이 보이면, 당황해서 `Ctrl + C`를 누르거나 터미널을 닫지 마세요. 오류 메시지 자체가 "어디서 왜 실패했는지" 알려 주는 가장 좋은 힌트입니다. 천천히 읽어 보고, 그대로 복사해 검색하면 대부분 해결 방법이 나옵니다. 강제 종료는 "진짜로 무한 반복에 빠진 것 같다"고 확신할 때만 마지막 수단으로 쓰세요.

:::

```bash
node -v                          # 20 이상인지 확인
claude --version                 # Claude Code CLI 설치 확인
npx paperclipai onboard --yes    # 설치 + 서버 실행
```

`onboard` 명령이 끝나면 브라우저에서 [http://localhost:3100](http://localhost:3100)이 자동으로 열리고, 첫 방문이라면 **온보딩 마법사**가 기다리고 있습니다.

![첫 화면 — 온보딩 마법사의 Company 단계](/screenshots/01-onboarding-company.png)

마법사의 네 단계(Company → Agent → Task → Launch)가 다음 장의 주제입니다. 지금 당장 눌러서 진행해도 되지만, 각 단계가 무엇을 의미하는지 이해하고 넘어가는 편이 훨씬 단단한 기반이 됩니다. 읽고 넘어오세요.

## 두 번째부터는 run

혹시 이게 헷갈리시나요? 괜찮습니다, 차근차근 풀어 드리겠습니다.

처음 설치 이후에 터미널을 닫으면 서버도 함께 종료됩니다. 다음 세션에서 PaperClip을 다시 쓰려면 `npx paperclipai run`으로 재시작해야 하지요. 여기서 `onboard`가 아니라 `run`이라는 점이 중요합니다.

`onboard`는 **새 인스턴스를 초기화하는 명령**입니다. 다시 실행하면 기존 데이터를 건드리지 않고 **별개의 새 인스턴스**를 생성합니다. 이 차이를 모르면 "어? 설치했는데 내 회사가 사라졌어요!" 같은 착각을 할 수 있습니다. 사실은 사라진 게 아니라, 두 번째 인스턴스로 접속하고 있어서 기존 데이터가 보이지 않을 뿐입니다.

## 내 AI 회사는 어디에 살고 있나

PaperClip 관련 데이터는 모두 `~/.paperclip/instances/default/` 아래에 저장됩니다. 이 폴더가 여러분 AI 회사의 "본사 건물"인 셈이지요.

이 폴더를 삭제하면 회사·에이전트·이슈·실행 기록이 전부 날아가고, 처음부터 다시 시작해야 합니다. 뒤집어 말하면, 문제가 생겼을 때 "깨끗하게 초기화하고 다시 시작"하려면 이 폴더를 지우고 `npx paperclipai onboard --yes`를 다시 실행하면 된다는 뜻입니다. 이 "리셋 기법"은 교재 중에 한 번쯤 쓰게 될지도 모르니 머릿속에 담아 두면 좋습니다.

다음 장에서는 바로 그 온보딩 마법사를 네 단계로 통과해 봅니다. 회사 이름 짓기 → 첫 에이전트 고르기 → 첫 태스크 정의 → 발사.
