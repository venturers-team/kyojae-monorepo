## 새 회사를 추가하는 방법

ClipHub에서 템플릿을 가져오는 명령은 모두 같은 패턴입니다. 좋은 소식이 있어요. 기존 회사를 덮어쓰지 않고 **새 회사로 추가**됩니다. 그래서 여러분의 첫 회사와 새 템플릿이 좌측 상단 드롭다운에 나란히 등장하지요.

언제든지 드롭다운에서 두 회사 사이를 오갈 수 있습니다. 여러 AI 회사를 한꺼번에 소유하는 셈이지요. 재밌지요?

```bash
# superpowers 템플릿 추가
npx companies.sh add paperclipai/companies/superpowers

# minimax-studio 템플릿 추가
npx companies.sh add paperclipai/companies/minimax-studio

# taches-creative 템플릿 추가
npx companies.sh add paperclipai/companies/taches-creative
```

Import가 끝나면 각 회사에도 지금까지 배운 패턴을 똑같이 적용하면 됩니다. 에이전트 어댑터를 Claude Code(또는 원하는 로컬 CLI)로 연결하고, Initiative를 하나 설정한 뒤, Run Heartbeat로 실행해 보세요.

재미있는 점이 있습니다. 회사마다 에이전트 구성과 스킬이 다르거든요. 그래서 **같은 Initiative를 줘도 전혀 다른 방식으로 태스크를 분해하는 모습**을 비교할 수 있습니다. 마치 같은 과제를 다른 팀에게 맡긴 것 같은 실험이 가능하지요.

## 추천 템플릿 세 가지

![ClipHub 추천 템플릿 — gstack · superpowers · minimax-studio](/infographics/04-templates.png)

### superpowers (4명, TDD 중심)

gstack보다 한 명 적고, 스킬도 14개로 더 간결합니다. CEO → Lead Engineer·Code Reviewer·Release Engineer의 1단계 계층이지요.

이 템플릿의 특징은 **TDD(Test-Driven Development, 테스트 주도 개발)** 사상을 중심에 두고 있다는 점입니다. "브레인스토밍 → 플랜 → 빌드 → 리뷰 → 쉽"의 직선적 흐름이 특징이지요.

gstack으로 한 바퀴를 돌고 나서 "조금 더 가벼운 템플릿에서 같은 실습을 해 보고 싶다" 싶다면, 이 템플릿이 적당합니다. 단순함의 미학을 맛볼 수 있지요.

### minimax-studio (5명, 비코딩 계열)

디지털 스튜디오 템플릿입니다. 앱 개발·VFX·문서 제작을 담당하는 5명의 에이전트로 구성되지요.

프로그래밍보다 **컨텐츠 제작** 쪽 작업을 에이전트에게 맡기고 싶을 때 선택합니다. 예를 들어 "블로그 포스트 10개 생성"이나 "앱 사용 설명서 제작" 같은 Initiative가 어울리지요.

gstack과 비교해 도메인이 완전히 다릅니다. 그래서 "같은 PaperClip 플랫폼이 얼마나 유연하게 다른 종류의 업무에 적용되는지"를 체감할 수 있지요. 플랫폼의 범용성을 확인하는 재미가 있습니다.

### taches-creative (6명, 메타 스킬)

사고 프레임워크와 AI 워크플로우 최적화에 특화된 템플릿입니다. 35개의 스킬이 대부분 "메타" 레벨이지요.

"메타"라는 게 무슨 뜻이냐면, 구체적인 결과물을 만드는 스킬이 아니라 "문제를 더 잘 정의하는 법", "워크플로우를 더 효율적으로 재구성하는 법" 같은 **상위 사고 도구들**입니다. 일하는 방식 자체를 다듬는 도구라고 할까요.

PaperClip으로 연구·기획·컨설팅 같은 작업을 해 보고 싶다면, 한 번 시도해 볼 만합니다. 생각하는 법을 AI와 함께 체계화해 볼 수 있거든요.

## 세 갈래의 업그레이드 경로

교재는 다음 단계 경로를 세 가지로 제안합니다. 어느 경로가 여러분에게 맞을지 한 번 고민해 보세요.

**첫 번째 경로는 gstack에 머무르는 방향입니다.** 27개 스킬을 하나씩 깊이 파고드는 길이지요. 실제 소프트웨어 개발 프로젝트에 적용하기에 가장 실전적입니다. "한 우물을 깊게 파기" 전략입니다.

**두 번째 경로는 다른 템플릿으로 병렬 확장하는 방향입니다.** 바로 위에서 소개한 템플릿들로 도메인을 넓히는 길이지요. PaperClip의 범용성을 체감하는 데 좋습니다.

**세 번째 경로는 다음 페이지에서 다룰 직접 커스터마이징입니다.** 기존 템플릿을 쓰지 않고, 자신의 문제에 맞는 에이전트 조직을 처음부터 설계하는 길이지요. 가장 용기 있는 길이고, 가장 깊이 있게 배우는 길이기도 합니다.

## 정답은 없습니다

어느 경로든 정답은 없습니다. PaperClip은 "올바른 회사 구조"를 강제하지 않는 플랫폼이지요.

여러분의 문제에 맞춰 자신만의 방식을 찾아가는 것이 이 도구의 본질입니다. 이 자유로움이 조금 막막하게 느껴질 수도 있지요. 하지만 그 자유야말로 PaperClip이 주는 가장 큰 선물입니다.
